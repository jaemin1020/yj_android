# yj_android - Android 및 웹 애플리케이션 개발 프로젝트

## 프로젝트 개요

이 프로젝트는 Android 모바일 애플리케이션과 Java 기반 웹 애플리케이션을 개발하며, 다양한 기술 스택과 데이터베이스 통합을 경험한 개인 포트폴리오 작품입니다. Android 개발의 기초부터 웹 백엔드 구축, 다중 데이터베이스 지원까지 폭넓은 개발 경험을 쌓기 위해 진행되었습니다.

프로젝트를 통해 모바일과 웹 플랫폼의 차이점을 이해하고, Gradle과 Maven 같은 빌드 도구의 활용법을 익혔습니다. 또한, 네트워크 통신, 데이터 처리, MVC 패턴 적용 등 실무적인 개발 기술을 적용해 보았습니다.

## 프로젝트 구성

### 1. Final1901175 - Android 모바일 애플리케이션

**개발 기간**: 2023년 10월 - 2023년 11월

**역할**: 개인 실습 프로젝트 (풀스택 개발)

**기술 스택**:

- 언어: Java 8
- 프레임워크: Android SDK
- 빌드 도구: Gradle
- 라이브러리: Volley (네트워크), Gson (JSON 파싱), AndroidX (UI 컴포넌트)

**주요 기능 및 구현**:

- **네트워크 통신**: Volley 라이브러리를 사용하여 RESTful API와의 데이터 교환 구현. 비동기 요청 처리로 앱의 반응성을 향상시켰습니다.
- **데이터 처리**: Gson을 활용한 JSON 데이터의 직렬화/역직렬화. API 응답을 효율적으로 파싱하여 UI에 표시.
- **UI 설계**: Material Design 가이드라인 준수. ConstraintLayout을 사용하여 다양한 화면 크기에 대응하는 반응형 인터페이스 구축.
- **SDK 호환성**: 최소 SDK 29 (Android 10)부터 지원하여 최신 보안 및 기능 활용.

**배운 점**:

- Android 생명주기 관리와 메모리 누수 방지 기법 습득.
- Gradle 빌드 최적화로 빌드 시간을 단축하는 방법 학습.
- 모바일 앱의 사용자 경험(UX) 설계 중요성 인식.

**스크린샷**: x

### 2. newmyHomepage - Java 웹 애플리케이션

**개발 기간**: 2023년 10월 - 2023년 11월

**역할**: 개인 실습 프로젝트 (백엔드 개발)

**기술 스택**:

- 언어: Java
- 프레임워크: Spring Framework 4.3.25, eGovFramework RTE 3.10.0
- 빌드 도구: Maven
- 데이터베이스: Altibase, Cubrid, MySQL, Oracle, Tibero (다중 DB 지원)
- 서버: Servlet API 2.5

**주요 기능 및 구현**:

- **MVC 아키텍처**: Spring MVC를 기반으로 한 모델-뷰-컨트롤러 패턴 적용. 코드의 모듈성과 유지보수성 향상.
- **데이터 액세스**: eGovFramework의 PSL (Persistence Service Layer)을 사용하여 데이터베이스 연동. 다중 데이터베이스 지원으로 유연한 배포 환경 구축.
- **보안 및 유틸리티**: ID 생성기, 속성 파일 관리, 문자열 처리 유틸리티 등 프레임워크 제공 기능 활용.
- **웹 배포**: WAR 파일로 패키징하여 Tomcat 등의 서블릿 컨테이너에 배포 가능.

**배운 점**:

- 엔터프라이즈급 Java 웹 개발의 구조적 접근법 습득.
- 다중 데이터베이스 환경에서의 쿼리 최적화와 호환성 관리 경험.
- Maven 의존성 관리와 빌드 라이프사이클 이해.

**데이터베이스 스크립트**: 프로젝트에 포함된 DDL 및 데이터 스크립트로 다양한 DB 환경 테스트 가능.

## 기술적 도전과 해결

- **다중 플랫폼 지원**: Android와 웹 프로젝트를 하나의 리포지토리에서 관리하며, 버전 관리와 CI/CD 파이프라인 구축 경험.
- **호환성 이슈**: 다양한 Android 버전과 데이터베이스에서 발생하는 호환성 문제를 해결하기 위해 조건부 컴파일과 폴백 메커니즘 적용.
- **성능 최적화**: Volley의 캐싱 기능과 데이터베이스 인덱싱을 통해 로딩 시간을 개선.

## 설치 및 실행 가이드

### Android 앱 실행

1. Android Studio 설치 및 프로젝트 임포트.
2. Gradle Sync 수행.
3. 에뮬레이터 또는 디바이스에서 디버그 모드로 실행.

### 웹 애플리케이션 배포

1. JDK와 Maven 설치.
2. `mvn clean install` 명령으로 빌드.
3. 생성된 WAR 파일을 웹 서버에 배포.
4. DATABASE 폴더의 스크립트로 DB 초기화.

## 향후 개선 방향

- Android 프로젝트에 Kotlin 마이그레이션으로 현대적인 코드베이스 구축.
- 웹 프로젝트에 Spring Boot 도입으로 마이크로서비스 아키텍처 적용.
- CI/CD 파이프라인 구축으로 자동화된 테스트와 배포 구현.
